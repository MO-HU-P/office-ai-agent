# Office AI Agent

**🇯🇵 日本語** | [🇬🇧 English](README.md)

**AIに話しかけるだけで、Word・Excel・PowerPointの書類を作ってくれるアプリです。**

「月ごとの売上表を作って」「議事録のひな形を用意して」のようにチャットでお願いすると、AIがその場でファイルを作成・編集します。できあがったファイルは、いつものWordやExcel、PowerPointでそのまま開けます。

## 画面の見かた

アプリは左右2つに分かれています。

- **左側**：AIとの会話スペース。ここにお願いごとを入力します。
- **右側**：AIが作った書類のプレビュー。AIが作業すると、その内容が自動でここに表示され、リアルタイムで更新されていきます。

チャットで指示すると、AIが「表を作成」「文字を太字に」といった作業を1つずつ実行し、その様子が左側に表示されます。作業が終わると右側に完成した書類が現れます。

画面右上の**歯車アイコン（設定）**からは、使うAIを切り替えられます。クラウド／このパソコンの切り替え、AIモデルの選択・追加・削除、AIの考え方（速さ重視か品質重視か）の変更が、コマンド入力なしでできます。

## できること

チャットで話しかけるだけで、次のようなことができます。「例」の文はそのままコピーして入力できます。

| やりたいこと | 例（このまま入力できます） |
|---|---|
| **Excelの表づくり・集計**<br>表の作成、計算・集計、数式の入力、色や書式の設定、シートの追加。条件に合う行を探して色付けなども | 「月別売上の表を作って、合計行と色付けもして」<br>「売上が100万円を超える行に黄色を付けて」 |
| **Wordの文書作成・編集**<br>見出し・箇条書き・表のある文書。特定の言葉のある段落を探して直したり、何箇所もまとめて修正したり | 「会議の議事録テンプレートをWordで作って」<br>「この文書の言い回しを全体的にやわらかくして」 |
| **お手本の文書と同じ体裁に**<br>既存のWord文書からフォント・色・見出しの体裁を読み取って、新しい文書や別の文書にそっくり適用 | 「この文書と同じ体裁で、新しい報告書を作って」 |
| **文書の校閲（変更履歴で提案）**<br>本文を勝手に書き換えず、元の文に取り消し線・直した文に下線を付けた「見え消し」で提案します。Wordで開けば1つずつ承認／却下できます（Word文書向け） | 「この文書を校正して、直しは変更履歴（見え消し）で提案して」 |
| **コメントで指摘**<br>本文は変えずに、気になる点を吹き出しコメントで残します（Word文書向け） | 「本文は直さず、気になる箇所にコメントで指摘して」 |
| **文書の翻訳（Word・PowerPoint・Excel）**<br>文書を別の言語に翻訳します。原文を訳文で上書きするか、原文を残して訳を併記するかを、作業を始める前にAIが確認します。数値・日付やExcelの数式はそのまま保たれます | 「この文書を英語に翻訳して」<br>「資料.pptx を日本語に翻訳して、原文を残して併記して」 |
| **2つのファイルの違いを調べる**<br>新旧2つのファイルを見比べて、どこが変わったかをやさしい日本語で説明します（Word・Excel・PowerPoint） | 「企画書_旧.docx と 企画書.docx の違いを教えて」 |
| **編集の確認と巻き戻し（自動バックアップ）**<br>ファイルを変更・削除するたびに、直前の状態が自動で保存されます。AIの編集で「どこが変わったか」を確認したり、うまくいかなかった編集を取り消したりできます | 「企画書.docx をさっきの編集前に戻して」<br>（プレビュー右上の 🕘 ボタンからも確認・巻き戻しできます） |
| **個人情報のマスク（匿名化）**<br>メールアドレス・電話番号・URL・郵便番号などを自動で伏せた別ファイルを作ります。元のファイルはそのまま残ります | 「このファイルの個人情報を匿名化して」 |
| **PowerPointのスライド作成・編集**<br>構成づくりから複数スライドの手直しまで。図形・画像・表・グラフの追加、いらない図形の削除や位置・大きさの調整も | 「"生成AIの業務活用" について5枚のスライドを作って」<br>「スライド3に月別売上の棒グラフを入れて」 |
| **スライドの見た目をAIが自分で確認**<br>設定で「画像対応」と表示されたAIモデルを選ぶと、AIがスライドを画像にして自分の目で見ながら、文字のはみ出しや図形の重なりを直します | 「スライドの見た目を確認して、崩れていたら直して」 |
| **定型文書の量産（差し込み）**<br>文書に `{{宛先}}` のような目印を入れておくと、値を差し替えた完成版を何通でも作れます。請求書・案内状・賞状などに便利 | 「案内状テンプレート.docx の {{宛先}} を "株式会社サンプル" にして完成版を作って」 |
| **データ分析・統計解析**<br>集計はもちろん、平均の差の検定から回帰分析・分散分析（ANOVA）・時系列予測といった本格的な統計モデリングまで。実験データの解析結果をそのままレポートにできます | 「このExcelの2つのクラスの点数に差があるか検定して」<br>「売上を気温と広告費で回帰分析して、結果をWordのレポートにまとめて」 |
| **グラフ入りレポートの作成**<br>データからグラフ（日本語ラベル対応）を描いて、WordやPowerPointに貼り付けたレポートを自動作成 | 「売上データを棒グラフにして、Wordのレポートにまとめて」 |
| **文書の仕上がりチェック**<br>書き忘れの見出し、Excelの計算式のエラー、スライドの詰め込みすぎなどをAIが点検 | 「この資料に問題がないかチェックして」 |
| **手持ちファイルの読み込み・分析・編集**<br>アップロードしたWord・Excel・PowerPoint・CSVをAIが読んで作業。パソコン内の元のファイルは変更されません | 「data.csv を分析して、結果をExcelにまとめて」 |
| **ファイルの整理**<br>コピー（バックアップ）作成、名前の変更、削除もチャットで | 「このファイルをコピーしてから編集して」<br>「企画書.docx の名前を 企画書_最終版.docx に変えて」 |

## 使い方の例

左側のチャットに、こんなふうに入力してみてください。

- 「月別売上のサンプルExcelを作って、合計行と色付けもして」
- 「"生成AIの業務活用" について5枚のスライドを作って」
- 「会議の議事録テンプレートをWordで作って」
- 「data.csv を分析して、結果をExcelにまとめて」
- 「実験データ.csv の測定値を条件Aと条件Bで回帰分析して、係数やp値の表と説明を入れたWordレポートを作って」

統計の専門知識がなくても大丈夫です。検定・回帰分析・分散分析（ANOVA）などの結果は、AIが表にまとめて意味をやさしい日本語で説明します。学生のレポートや研究データのとりまとめにも使えます。

右側にファイルを表示しているときは、入力欄のすぐ上に **「✍️校正」「📝要約」「🕶️匿名化」** のワンタッチボタンが出ます。押すだけで、表示中のファイルに対してその作業をお願いできます（自分で文章を入力しなくてもOKです）。

プレビュー右上には **🕘（変更箇所を確認・元に戻す）ボタン** もあります。AIがそのファイルをどう変えたかを色分けの差分（赤＝削除・緑＝追加）で確認でき、「この変更前に戻す」を押せば編集前の状態に巻き戻せます。

自分のパソコンにあるファイルをアプリに読み込ませて編集してもらうこともできます。右側の上部にあるアップロードボタン（↑のアイコン）から追加してください。

> **アップロードするファイルの中身にご注意ください。** クラウドのAI（Ollama Cloud・OpenAI・Gemini）に作業を頼むと、そのファイルの中身はAIの提供元のサーバーに送られます。**氏名・連絡先などの個人情報や、社外秘の資料・未公開情報といった秘匿性の高い（機密の）情報を含むファイルは、そのままアップロードしないことをおすすめします。** どうしても個人情報を扱う必要があるときは、先に「匿名化」（後述）で伏せてからアップロードできます。ただし匿名化で自動的に伏せられるのは決まった形式の個人情報だけで、機密情報そのものは伏せられません。機密性の高い書類を扱うときは、設定で「このパソコン（ローカル）」のAIに切り替えてお使いください（ローカルなら中身は外部に送られません）。

**音声でも指示できます。** 入力欄の右にあるマイクボタン（🎤）を押して話すと、話した内容がそのまま文字になります。もう一度押すと停止します。初回はブラウザが「マイクの使用を許可しますか？」と聞いてくるので「許可」を選んでください。

> 音声入力はブラウザ内蔵の音声認識（Chrome・Edgeで利用可能）を使うため、音声はブラウザ提供元のサーバーで文字に変換されます。機密性の高い内容は、これまでどおりキーボードで入力してください。なお、`http://localhost:3000` 以外のアドレス（他のパソコンからのアクセス等）では、ブラウザの仕様によりマイクが使えません。

作られたファイルは、このアプリのフォルダ内の `workspace` という場所に保存されます。パソコンから直接開くこともできます。

## 作ったファイルを開くには

AIが作るのは、いつものWord・Excel・PowerPointと同じ形式のファイルです。仕上げや印刷は、お使いのパソコンのアプリで開いて行うのがおすすめです。

- **Windows / Mac の方**：お使いの **Microsoft Office**（Word・Excel・PowerPoint）でそのまま開けます。これが基本の使い方です。
- **Linux の方**：Microsoft Officeが使えないため、**LibreOffice**（無料で使えるOffice互換ソフト）で開けます。

どのパソコンでも、右側のプレビューで内容を確認しながら作業を進められます。

# セットアップ（初回のみ）

初めて使うときは、下の**準備1〜4**をひととおり行ってから、アプリを起動します。一度準備すれば、次回からは起動コマンドだけでOKです。パソコンの操作に慣れていない方でも、順番どおりに進めれば大丈夫です。

## 準備1：Docker Desktop（ドッカー・デスクトップ）をインストールする

このアプリは **Docker Desktop** という無料のソフトの上で動きます。まずこれを入れます。

1. 公式サイト [https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/) を開きます。
2. お使いのパソコンに合わせてダウンロードボタンを押します。
   - **Windows** の方：「Download for Windows」を選びます。
   - **Mac** の方：「Download for Mac」を選びます（画面に **Apple Chip**（M1/M2/M3など新しいMac）と **Intel Chip**（古いMac）の選択が出たら、お使いのMacに合う方を選びます。分からなければ、画面左上のリンゴマーク →「このMacについて」で確認できます）。
3. ダウンロードしたファイルを開き、画面の指示に従ってインストールします（Windowsは何度か「はい」を押すだけ、Macはアイコンを「Applications」フォルダにドラッグするだけです）。
4. インストールが終わったら **Docker Desktop を起動**します。少し待って、画面のクジラのマーク 🐳 が動かず落ち着いた状態になれば準備完了です。

> **大切**：このアプリを使う間は、**Docker Desktop を起動したままにしておいてください**（閉じるとアプリも止まります）。パソコン再起動後などは、先に Docker Desktop を立ち上げてください。

## 準備2：ターミナル（コマンドを入力する画面）を開く

「ターミナル」は、パソコンに文字で命令を伝えるための画面です。難しく見えますが、やることは**コマンドをコピー＆貼り付けして Enter を押すだけ**です。

- **Windows** の方：画面下のスタートメニューを開き、「**PowerShell**」と入力して、出てきた「Windows PowerShell」をクリックします。
- **Mac** の方：`⌘`（コマンド）＋ スペースキーを押して、「**ターミナル**」と入力し、出てきた「ターミナル」を開きます。

黒い（または白い）画面が出れば成功です。

## 準備3：アプリのフォルダへ移動する

ターミナルに、この `office-ai-agent` フォルダの場所を教えます。次のように操作します。

1. ターミナルに **`cd `**（cd のあとに半角スペース）と入力します（まだ Enter は押しません）。
2. `office-ai-agent` フォルダを、そのまま**ターミナルの画面の上にドラッグ＆ドロップ**します。すると、フォルダの場所（住所のようなもの）が自動で入力されます。
3. **Enter キー**を押します。

これで、ターミナルがアプリのフォルダの中にいる状態になりました。ここから起動の準備に進みます。

## 準備4：AIの頭脳（モデル）を選ぶ

このアプリは、2通りの方法でAIを動かせます。

| 方法 | 特徴 | こんな人に |
|---|---|---|
| **クラウド** | インターネット経由で高性能なAIを使う。事前の準備が少なく、賢い。 | まず試したい人・高い品質がほしい人（推奨） |
| **ローカル** | 自分のパソコンの中だけでAIを動かす。インターネット不要で、情報が外に出ない。 | 機密情報を扱う人・オフラインで使いたい人 |

どちらを使うかは、起動後にアプリ画面右上の**設定（歯車アイコン）**からいつでも切り替えられます。最初は **クラウド** がおすすめです。

> **OpenAI・Google GeminiのAPIキーをお持ちの方へ（任意）**：上の2つ（Ollama）に加えて、**OpenAI** や **Google Gemini** のAIも選べます。設定方法は下の「[OpenAIを使う場合](#openaiを使う場合任意)」「[Google Geminiを使う場合](#google-geminiを使う場合任意)」を参照してください。お持ちでない方は気にせず、クラウド（Ollama）で始めてください。

## クラウドで使う場合（推奨）

1. [https://ollama.com/settings/keys](https://ollama.com/settings/keys) にアクセスして「APIキー」（アプリ用の合言葉のようなもの）を発行します。
2. アプリのフォルダにある見本ファイル `.env.example` をコピーして、`.env` という名前のファイルを作ります（ターミナルなら次のコマンド。Windowsは `cp` の代わりに `copy`）。

   ```bash
   cp .env.example .env
   ```

3. 作成した `.env` を「メモ帳」や「テキストエディット」などで開き、発行したキーを貼り付けて保存します。

   ```
   OLLAMA_API_KEY=ここに発行したキーを貼り付け
   ```

4. 準備2〜3で開いたターミナル（アプリのフォルダに移動済み）に、次のコマンドをコピーして貼り付け、**Enter** を押します。

   ```bash
   docker compose up -d --build
   ```

   > 初めての起動は、必要な部品のダウンロードと組み立てで**数分かかります**。文字がたくさん流れますが、故障ではありません。画面が動いている間はそのまま待ってください。

5. 「done」などと表示されて入力できる状態に戻ったら、ブラウザで **http://localhost:3000** を開きます。
6. 画面右上の表示が緑色（例：「Ollama Cloud · gpt-oss:120b」）になれば準備完了です。あとはチャットに話しかけるだけ。

アプリを止めたいときは、同じターミナルで `docker compose down` を実行します。

## ローカル（このパソコン）で使う場合

まずクラウドの手順4と同じように `docker compose up -d --build` でアプリを起動します（APIキーは不要です）。そのあと、画面右上の**設定（歯車アイコン）**を開いて、

1. 「AIの実行場所」で「**このパソコン**」を選ぶ
2. 「使用するAIモデル」で使いたいモデルを選ぶ（未ダウンロードなら入力欄から**ダウンロード**。数GBあるため、回線速度によっては数十分かかります）
3. 「保存」を押す

これだけで切り替わります。ダウンロードが終われば、以降はインターネットなしで使えます。不要になったモデルは、設定画面のゴミ箱アイコンで削除してディスクの空きを増やせます。

## OpenAIを使う場合（任意）

OpenAIのAPIキーをお持ちの方は、**OpenAI** のAIも使えます（お持ちでない方は、この節はとばして構いません）。Ollamaのクラウドが混み合って途中で止まりやすいときに、より安定した選択肢になります。

1. [https://platform.openai.com/api-keys](https://platform.openai.com/api-keys) でAPIキーを発行します。
2. アプリのフォルダにある `.env` ファイルを開き、次の行にキーを貼り付けて保存します（`.env` がまだ無ければ、クラウドの手順と同じように `.env.example` からコピーして作ります）。

   ```
   OPENAI_API_KEY=ここに発行したキーを貼り付け
   ```

3. ターミナルで `docker compose up -d` を実行して起動し直します（`.env` の変更を反映するためです）。
4. ブラウザで画面右上の**設定（歯車アイコン）**を開くと、「AIの実行場所」に「**OpenAI**」が追加されています。選んで、使うモデル（`gpt-5.4-mini` など）を選び、「保存」を押せば切り替わります。

> - キーを設定していないときは、設定画面に「OpenAI」は**表示されません**（Ollamaだけのすっきりした画面のままです）。
> - より正確な文書がほしいときは、一覧の `gpt-5.4-mini` を選ぶと仕上がりが安定します（さらに上位が必要なら入力欄から `gpt-5.4` などを追加）。`gpt-4o-mini` は速くて安価ですが、複雑な指示では体裁が崩れることがあります。
> - 一覧に無いモデル名も入力欄から自由に追加できます（新しいモデルが出たときなど）。追加したモデルは次回も一覧に残り、不要になったら各行の削除ボタン（🗑️）で消せます（もとからある候補は消せません）。
> - モデル名の欄にAPIキー（合言葉）をうっかり貼り付けても、キーとして保存されないよう自動で弾かれます。
> - OpenAIの利用にはOpenAI側の料金がかかります（Ollamaの無料枠とは別です）。

## Google Geminiを使う場合（任意）

GoogleのGemini APIキーをお持ちの方は、**Google Gemini** のAIも使えます（お持ちでない方は、この節はとばして構いません）。OpenAIと同じく、Ollamaのクラウドが混み合って途中で止まりやすいときの、より安定した選択肢になります。

1. [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey) でAPIキーを発行します。
2. アプリのフォルダにある `.env` ファイルを開き、次の行にキーを貼り付けて保存します（`.env` がまだ無ければ、クラウドの手順と同じように `.env.example` からコピーして作ります）。

   ```
   GEMINI_API_KEY=ここに発行したキーを貼り付け
   ```

3. ターミナルで `docker compose up -d` を実行して起動し直します（`.env` の変更を反映するためです）。
4. ブラウザで画面右上の**設定（歯車アイコン）**を開くと、「利用するサービス」に「**Google Gemini**」が追加されています。選んで、使うモデル（`gemini-3.1-flash-lite` など）を選び、「保存」を押せば切り替わります。

> - キーを設定していないときは、設定画面に「Google Gemini」は**表示されません**（Ollamaだけのすっきりした画面のままです）。
> - はじめは軽くて速い `gemini-3.1-flash-lite` がおすすめです。より品質がほしいときは `gemini-2.5-flash` を選ぶと安定します。一覧に無いモデル名も入力欄から自由に追加でき（新しいモデルが出たときなど）、追加分は各行の削除ボタン（🗑️）で消せます（もとからある候補は消せません）。
> - モデル名の欄にAPIキー（合言葉）をうっかり貼り付けても、キーとして保存されないよう自動で弾かれます。
> - Geminiの利用にはGoogle側の料金がかかる場合があります（無料枠の有無・上限はGoogleの規定によります）。「アクセスが拒否されました」と表示される場合は、[Google AI Studio](https://aistudio.google.com/apikey) でキーが有効か、地域や課金設定に制限がないかをご確認ください。

## 2回目以降の起動（かんたん）

一度セットアップしたあとは、準備1〜4をやり直す必要はありません。次の3つだけでOKです。

1. **Docker Desktop** を起動する（クジラのマーク 🐳 が落ち着くまで待つ）。
2. ターミナルを開き、`cd ` のあとに `office-ai-agent` フォルダをドラッグして Enter（準備3と同じ）。
3. 次のコマンドを実行する。

   ```bash
   docker compose up -d
   ```

そのあとブラウザで **http://localhost:3000** を開けば使えます。止めるときは `docker compose down` です。

> パソコンやフォルダの場所を毎回覚えるのが大変なら、上の3行を書いたメモを手元に残しておくと便利です。

# 設定の説明

## 設定画面（歯車アイコン）— ふだんはこちら

AIに関する設定は、アプリ画面右上の歯車アイコンから変更するのがいちばん簡単です。保存するとすぐに反映され、再起動もコマンド入力も不要です。

| 項目 | 意味 |
|---|---|
| AIの実行場所 | 「Ollama クラウド」（高性能・APIキー必要）か「このパソコン」（データが外に出ない）を選びます。OpenAIのAPIキーを設定している場合は「OpenAI」も選べます |
| 使用するAIモデル | 使うAIを一覧から選びます。「画像対応」と表示されたモデルは、AIがスライドや文書の見た目を画像で確認しながら修正できます。一覧に無いモデルは入力欄から追加できます（ローカルはダウンロード）。不要なモデルの削除も可能です（ローカルはディスクの節約、OpenAIは自由入力で追加した候補の削除） |
| 回答の考え方 | 「じっくり考える」ほど高品質ですが返答が遅くなります。基本は「自動」のままでOK |

設定画面で保存した内容は `data/settings.json` に記録され、下記の `config.toml` より優先されます。`config.toml` の値に戻したいときは `data/settings.json` を削除してください。

> **安全のため**：APIキー（合言葉）だけは、この画面では表示も変更もできません。`.env` ファイルでのみ管理します。

## config.toml（AIの動作設定のデフォルト値・上級者向け）

変更したら `docker compose restart backend` を実行すると反映されます。

| 項目 | 意味 |
|---|---|
| `llm.provider` | 使うAIの提供元。`ollama`（既定）か `openai`。ふだんは設定画面から切り替えるのが簡単です |
| `llm.mode` | Ollama利用時に `cloud`（クラウド）か `local`（ローカル）を選びます |
| `llm.model` | 使うAIモデルの名前。クラウドなら `gpt-oss:120b` など、ローカルなら `qwen3.5:9b` や `gemma4:12b` など、OpenAIなら `gpt-5.4-mini` など |
| `llm.openai_models` | OpenAI利用時に設定画面の一覧へ出すモデル候補（例 `["gpt-5.4-mini", "gpt-4o-mini"]`）。モデルが廃止・追加されたらここを直せば一覧を差し替えられます（先頭が既定）。一覧に無いモデルも設定画面から自由に追加できます |
| `llm.reasoning` | AIの「じっくり考えるモード」。基本は `auto` のままでOK |
| `llm.num_ctx` | AIが一度に覚えていられる情報量（ローカル利用時のみ）。基本はそのままでOK |
| `agent.max_steps` | AIが1回の依頼でこなす作業の上限回数 |
| `agent.llm_idle_timeout` | AIの返事が止まったとき、何秒待ってから自動でやり直すか（初期値180秒）。基本はそのままでOK |

# 知っておくとよいこと

- **作った書類はちゃんと保存されます。** AIが行った作業（表の作成や書式の変更など）は、その都度あなたのパソコンの `workspace` フォルダに書類ファイルとして保存されます。アプリを閉じても、パソコンを再起動しても、Docker が突然止まっても、**でき上がった書類は消えません**（次回そのまま開けます）。保存は「いったん別ファイルに書いてから置き換える」安全な方式で行うため、万一保存中にアプリが落ちても、元の書類が壊れることはありません。
  - ※ ファイルの中身は残りますが、**チャットの会話のやりとりは保存されません**。ブラウザの画面を開き直したり、アプリを再起動したりすると会話は新しく始まりますが、書類はそのまま残っているので「このファイルを開いて〜して」と続きを頼めます。
- **AIの編集は、あとから確認して元に戻せます。** ファイルが変更・削除されるたびに、直前の状態がアプリ内（`workspace` の `.history` フォルダ）へ自動でバックアップされます（1ファイルにつき直近20世代まで）。プレビュー右上の🕘ボタンで「直前の編集でどこが変わったか」（Wordは段落、Excelはセル、PowerPointはスライドの文章単位）を確認でき、「この変更前に戻す」で巻き戻せます。チャットで **「◯◯を元に戻して」** と頼んでもOKです。巻き戻す前の状態も保存されるので、**巻き戻し自体もあとから取り消せます**。※差分に表示されるのは文章・セルの値の変更だけで、色やフォントなど書式だけの変更は表示されません（巻き戻せば書式も含めて戻ります）。
- **アップロードした元のファイルは編集されません。** ファイルをアップロードすると、その**コピー**がアプリ（`workspace` フォルダ）に取り込まれ、AIが読み書きするのはこのコピーだけです。パソコンの元の場所にあるファイルには一切手を付けないので、大切なファイルも安心して任せられます。
  - ※ 取り込んだコピーのほうは、編集を頼むとその場で上書きされますが、上で説明したとおり**直前の状態が自動でバックアップされる**ので、「元に戻して」と頼めば編集前に戻せます。編集前の版を別ファイルとして手元に残したいときは、**「このファイルをコピーしてから編集して」** と頼んでください。書式ごとそっくりな複製を作ってから作業してくれます。**「ファイル名を◯◯に変えて」** のような名前の変更もチャットで頼めます。
- **大きな依頼は、いくつかに分けて頼むのがコツです。** 「表を作って、統計解析もして、レポートにもまとめて」のように一度にたくさん頼むと、作業が長くなり途中で失敗しやすくなります。「①このデータで表を作って」→（できたのを確認して）→「②統計解析を追加して」→「③結果をWordのレポートにまとめて」のように**1つずつ順番に**頼むと確実で、途中の様子も確認しながら進められます。
- **途中でエラーが出ても、作りかけの書類は消えません。** クラウドのAIが混み合っているときなどに「⚠️ AIモデルの呼び出しに失敗しました」と表示されることがあります。少し待ってから **「◯◯（ファイル名）が途中で止まっているので、続きを完成させて」** と頼めば、そこから再開できます。AIの返事が長時間止まってしまった場合も、アプリが自動で打ち切ってやり直すので、そのまま少しお待ちください。OpenAIのAPIキーをお持ちなら、設定で **OpenAI** に切り替えると、長いレポートも安定して最後まで作れることがあります。
- **AIモデルによって賢さが変わります。** うまく書類が作れないときは、より高性能なモデル（クラウドの `gpt-oss:120b` や、ローカルなら `gpt-oss:20b`・`gemma4:12b` など）を試してください。小さいモデルは、複雑な指示でうまく動かないことがあります。
- **モデルの一覧には「書類作成向きでないもの」も含まれます。** クラウドのモデル一覧は Ollama Cloud が提供している全モデルをそのまま表示しています。名前に `coder`・`code`・`devstral` などが付くモデルは**プログラミング専用**のため、書類作成にはおすすめしません。迷ったら `gpt-oss:120b` のような汎用モデルを選んでください。
- **クラウドのモデルは提供終了（廃止）されることがあります。** 使用中のモデルが提供終了すると、画面右上が黄色の表示（「モデル ○○ は利用できません」）に変わり、チャット画面にも案内が出ます。その場合は設定（歯車アイコン）を開いて、一覧から別のモデルを選んで保存すれば、すぐに使えるようになります。
- **クラウドには、無料プランでは使えないモデルもあります。** 一覧から選ぶこと自体はできますが、チャットを送ると「お使いのプランでは利用できません」という案内が表示されます。その場合は、設定から別のモデルに切り替えるか、Ollama の有料プラン（[ollama.com](https://ollama.com)）へのアップグレードをご検討ください。
- **個人情報を「AIに送る前に」自分で伏せておくための機能です。** クラウドのAI（Ollama Cloud・OpenAI・Gemini）に作業を頼むと、書類の中身はAIの提供元のサーバーに送られます。氏名や連絡先などを含んだままAIに渡したくないときは、**先にこの「匿名化」で個人情報を伏せ、でき上がった匿名化ファイルのほうをAIに読ませて作業を頼む**、という進め方ができます。どの書類を匿名化するか（そもそも匿名化するかどうか）は、**あなたご自身の判断で**決めてください（アプリが勝手に判断することはありません）。
  - **自動で伏せられるもの**：メールアドレス・電話番号・URL・郵便番号・マイナンバー・クレジットカード番号のように、**形式が決まった個人情報**。この置き換えは**AIに中身を送らず、アプリの中だけ**で行います。でき上がるのは「(匿名化)◯◯」という別ファイルで、元のファイルはそのまま残ります。
  - **自動では伏せられないもの**：氏名・住所・会社名や学校名のように、決まった形の無い情報は、取りこぼしを防げないため自動マスクの対象外です。これらが含まれる書類は、でき上がった匿名化ファイルを**目視でも確認**してください。
- **ファイルの「違いを調べる」機能について。** 2つのファイルの**文章やセルの値**を見比べて、変わった点をやさしい日本語で説明します。色やフォントなどの見た目の違いは対象外です。Excelで列を追加したり行を大きく並べ替えたりした場合は、変更点が実際より多く見えることがあります（新旧の版を見比べるといった使い方が得意です）。
- **Excelの計算式について。** AIが `=SUM(...)` のような計算式を入れた場合、プレビュー上では式のまま表示されることがあります。実際にExcelで開くと正しく計算されます。
- **安全のための注意。** このアプリは、AIがパソコン上でプログラムを実行してデータ処理を行う機能を持っています。信頼できる環境（自分のパソコンや社内ネットワーク内）でのみお使いください。

# 補足：エンジニア向け情報

技術構成：フロントエンドは React + TypeScript、バックエンドは Python（FastAPI）。AIエージェントは LangChain + Ollama で動き、ファイル操作には python-docx / openpyxl / python-pptx を使用。プレビューは Excel を独自グリッド、Word を docx-preview、PowerPoint を LibreOffice 経由の画像変換で表示しています。全体を Docker Compose で起動します。

校閲系ツール（`agent/tools/`）：変更履歴の提案（`word_suggest_edits`）は python-docx に API が無いため `w:ins`/`w:del` を直接生成し、docx-preview の `renderChanges`/`renderComments` で見え消し・コメントを表示します（Word専用）。差分比較（`doc_diff`）は各形式からテキストを抽出して `difflib` で比較。匿名化（`anonymize_file`）は正規表現による決定論的なマスクをローカルで完結させ、生の個人情報を LLM に渡しません（対象は形式の定まった構造化PIIのみ。氏名・住所等は非対象）。

自動バックアップ・巻き戻し（`services/history.py`）：全ツールとアップロードが通るアトミック保存（`atomic.py`）に1箇所フックし、既存ファイルの上書き・削除の直前の状態を `workspace/.history/<ファイル名>/` に世代保存します（1ファイル20世代、古いものから自動削除）。`run_python` はこの保存経路を通らずにファイルを書き換えうるため、実行前にワークスペース全体をバックアップします（未変更ファイルはサイズ+mtime比較でスキップされ軽量）。差分・巻き戻しの既定の単位は「ユーザーの1回の依頼（ターン）」で、REST は `GET /api/files/{name}/changes`（Excelはセル単位の差分）と `POST /api/files/{name}/restore`、エージェントツールは `restore_file` / `list_file_versions` です。巻き戻し前の状態も退避するため巻き戻し自体を取り消せます。`.history` を含む隠しパス（先頭が `.`）はツール・APIから操作できません。バックエンドのテストは `backend/tests` にあり、`docker compose run --rm --no-deps -v ./backend:/srv -w /srv backend sh -c "pip install -q pytest && python -m pytest tests"` で実行できます。

データ分析は `run_python` ツール経由で pandas / NumPy / SciPy / statsmodels / seaborn が使えます。statsmodels により回帰分析の詳細レポート（summary）・ANOVA・GLM・時系列（ARIMA）などの統計モデリングと、R風の式表現（`y ~ x1 + x2`）に対応しています。

既知の挙動メモ：モデルによっては（gpt-oss 等）、ツール呼び出しの合間に英語の思考テキストがチャットに表示されることがあります。`<think>` タグ付きの思考は除去していますが、タグ無しで出力される中間テキストはそのまま表示されます。

画像入力（vision）対応モデルを選択している場合は、エージェントがスライド/ページをPNG化して自分で見た目を確認する「見て直す」ループが自動で有効になります（対応可否は Ollama の `/api/show` の capabilities で判定）。テキスト専用モデルではこの機能は現れず、従来どおり動作します。

生成ファイルはホストの `./workspace` にバインドマウントで永続化され、コンテナやDockerが停止・削除されても失われません。各ツールの保存は一時ファイルへ書き出してから `os.replace()` で差し替えるアトミック保存で、保存中にプロセスが落ちても既存ファイルは破損しません。

Dockerを使わず開発する場合：

```bash
# バックエンド (Python 3.12推奨)
cd backend && pip install -r requirements.txt
WORKSPACE_DIR=../workspace OLLAMA_BASE_URL=http://localhost:11434 uvicorn app.main:app --reload

# フロントエンド
cd frontend && npm install && npm run dev   # http://localhost:5173
```

他のDockerプロジェクトと同時に動かしても衝突しないよう、プロジェクト名を `office-ai-agent` に固定し、外部に公開するのはポート 3000 のみにしています。GPUがないパソコンでローカルモデルを使う場合は、`docker-compose.yml` の `gpus: all` の行を削除してください。
